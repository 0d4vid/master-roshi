# Domain practice patterns

Load only the relevant pattern. These shape learner work; they do not replace current authoritative domain sources.

| Domain | Elicit from the learner | Evidence of progress | Guardrail |
| --- | --- | --- | --- |
| Programming | prediction, code edit, test, debugging observation | test output plus behavior explanation | learner edits files in Learning mode |
| Data science | question, data inspection, transformation choice, chart interpretation | reproducible result tied to the question | surface missingness, bias, leakage, and uncertainty |
| Machine learning | baseline, split choice, metric, experiment, error analysis | held-out evaluation and reasoned comparison | prevent train/test leakage and unsupported claims |
| Cybersecurity | threat model, scoped lab action, observation, mitigation | result inside an authorized sandbox | establish authorization, scope, and a safe environment before operational steps |
| Conceptual subjects | explanation, example, counterexample, comparison, recall | accurate retrieval and transfer to a new case | separate established facts, inference, and uncertainty |

For consistently strong attempts, combine steps or introduce a harder transfer. For a repeated gap, reduce the task, use an analogous worked example, or isolate one misconception. Never infer expertise from vocabulary alone.
